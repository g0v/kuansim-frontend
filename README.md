# Branch Rules
See [git-flow cheatsheet](http://danielkummer.github.io/git-flow-cheatsheet/)

*   All new code shall be push to `develop` branch
*   Release manager will merge `develop` to `master`

# Import fake data

check out fake data

```
$ make checkout
```

import to pgrest.

```
$ make import
```

# Start the server
```
$ npm i
$ ./script/server.sh
```
